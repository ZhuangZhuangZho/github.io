Vue 响应式四联问

周江豪 Lv1

第一问：Vue2在哪些情况操作数据不是响应式的？

1.给对象后续新增的 key 进行赋值。例如：给一个空对象，通过按钮新增一个属性，这个不是响应式的

2.删除对象中的某一项。例如：一个对象中有属性，通过按钮将这个属性删掉后，这个也不是响应式的

3.通过索引去修改数组的元素。例如：一个数组里面有3个值，通过点击按钮将下标为0的值进行修改，这个不是响应式的

第二问：为什么这些情况下不是响应式的？

1.因为Object.defineProperty( )是递归劫持的对象中的每一个属性，如果这个属性是后续被添加的，当然没有被劫持到

2.因为Object.defineProperty( )不支持对删除操作的劫持

3.并不是Object.defineProperty( )不支持对数组的劫持，而是作者出于对性能的原因没有这样做

第三问：怎么解决？

1.给对象后续新增的 key 进行赋值，解决方案：**Vue.set( )或this.$set( )或this.$forceUpdate ( ) **强制更新但不建议使用

2.删除对象中的某一项，解决方案：**Vue.delete( )或this.$delete( )或this.$forceUpdate ( ) **强制更新但不建议使用

3.通过索引去修改数组的元素，解决方案：**Vue.set( )或Vue.$set( )或arr.splice( )或this.$forceUpdate ( ) **强制更新但不建议使用

第四问：在Vue3中这些问题有没有得到解决？为什么可以解决？

1.Vu3响应式的原理从Object.defineProperty( )换成了Proxy，它可以直接代理整个对象，就无所谓这个属性是不是后续添加的了

2.Proxy支持对删除操作的拦截

3.Proxy对数组的操作不存在性能问题
